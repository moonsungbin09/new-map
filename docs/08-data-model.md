# 데이터 모델 (아주 쉽게)

이 문서는 "어떤 데이터를 저장할지"를 정합니다.

## 1) Place (장소)

| 필드 | 의미 | 필수/선택 | 예시값 |
|---|---|---|---|
| id | 장소 고유 번호 | 필수 | `plc_1001` |
| name | 장소 이름 | 필수 | `관악산 숨은 전망대` |
| latitude | 위도 | 필수 | `37.4451` |
| longitude | 경도 | 필수 | `126.9582` |
| level_type | 공간 유형(지상/지하/산) | 필수 | `산` |
| description | 장소 설명 | 선택 | `해질 때 뷰가 좋아요.` |
| creator_id | 등록한 사용자 ID | 필수 | `usr_2001` |
| created_at | 등록 시각 | 필수 | `2026-06-13T10:00:00Z` |

## 2) User (사용자)

| 필드 | 의미 | 필수/선택 | 예시값 |
|---|---|---|---|
| id | 사용자 고유 번호 | 필수 | `usr_2001` |
| email | 로그인 이메일 | 필수 | `mapmaker@example.com` |
| nickname | 화면에 보일 이름 | 필수 | `지도초보` |
| created_at | 가입 시각 | 필수 | `2026-06-13T09:30:00Z` |

## 3) ShareLink (공유 링크)

| 필드 | 의미 | 필수/선택 | 예시값 |
|---|---|---|---|
| id | 공유 링크 고유 번호 | 필수 | `sh_3001` |
| place_id | 어떤 장소를 공유하는지 | 필수 | `plc_1001` |
| token | 공유용 문자열 | 필수 | `ab12cd34` |
| expires_at | 만료 시각 | 필수 | `2026-06-20T10:00:00Z` |

## 4) 유효성 규칙
- `name`: 1~50자
- `latitude`: -90 ~ 90
- `longitude`: -180 ~ 180
- `level_type`: `지상`, `지하`, `산` 중 하나
