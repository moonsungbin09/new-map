# 아주 간단한 프론트 우선 배포 설계서

## 1. 목표
- 오늘 목표는 모바일 웹에서 다음 3가지만 명확하게 보이도록 만드는 것이다.
  1. 첫 화면(간단 소개)
  2. 장소 목록
  3. 장소 등록 버튼

## 2. 범위
### 포함
- 홈 화면 UI 단순화
- 기존 목록 API 연동 유지 (`GET /api/places`)
- 등록 버튼 이동 유지 (`/create.html`)
- 에러 문구 최소화

### 제외
- 등록 화면 구조 변경
- 상세 화면 구조 변경
- 인증/권한 기능 추가
- 고급 UI 애니메이션/디자인 리뉴얼

## 3. 변경 구조
- 백엔드/배포 구조는 그대로 유지한다.
- 프론트는 최소 파일만 수정한다.
  - `public/index.html`
  - `public/styles.css`
  - `public/app.js` (목록 로드/에러 문구 정도만 단순 정리)

## 4. 사용자 화면 흐름
1. 사용자가 첫 화면에 접속한다.
2. 상단에서 앱 목적과 등록 버튼을 본다.
3. 아래에서 장소 목록을 확인한다.
4. 등록 버튼을 누르면 `/create.html`로 이동한다.

## 5. 에러 처리
- 목록 조회 실패 시 한 줄 메시지만 표시한다.
  - 예: "목록을 불러오지 못했습니다."
- 화면이 복잡해지지 않도록 상세 디버그 정보는 노출하지 않는다.

## 6. 배포 및 완료 기준
- `main` 반영 후 Azure Web App 재배포
- 아래 3가지가 되면 1차 완료로 본다.
  1. 첫 화면이 바로 열린다.
  2. 목록이 표시된다(또는 실패 문구가 깔끔하게 표시된다).
  3. 등록 버튼이 `/create.html`로 이동한다.

## 7. 2차 보완(다음 단계)
- 등록/상세 화면 UX 정리
- 빈 목록 상태(데이터 없음) 문구 개선
- 시각 스타일 고도화
