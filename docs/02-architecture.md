# 아키텍처 (쉽게 이해하는 구조도 설명)

이 문서는 "앱이 안에서 어떻게 나뉘는지"를 설명합니다.

## 1) 큰 구조
- 화면(UI)
- 기능 처리(로직)
- 데이터 저장/불러오기

## 2) 데이터 흐름(간단)
1. 사용자가 화면에서 행동함
2. 앱이 필요한 데이터를 가져오거나 저장함
3. 결과를 화면에 다시 보여줌

## 3) 기술 선택
- Android: Kotlin
- UI: Jetpack Compose
- 로컬 저장: Room(간단 캐시/임시 저장)
- 서버 연동: REST API
- 배포/운영 기준: Azure 백엔드(API/DB) 연동을 전제로 설계

## 4) Azure 기준 실제 구성(쉬운 버전)
- 앱: Android 앱(사용자 입력, 화면 표시)
- API 서버: Azure App Service
- 데이터베이스: Azure SQL Database
- 인증: 이메일 로그인 + 서버가 발급한 토큰
- 데이터 흐름: 앱 -> API -> DB, 조회할 때는 DB -> API -> 앱
